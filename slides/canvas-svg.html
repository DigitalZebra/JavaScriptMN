Overview

What?

Canvas
  • New HTML tag part of HTML 5 specification
  • Provides low level raster based graphics API
SVG
  • Scalable Vector Graphics
  • XML file format which defines vector graphics
  • <SVG> tag can be embedded in HTML and rendered by browsers

Canvas

Quick facts:

  • Introduced by Apple in Webkit circa 2004
  • New element part of the HTML5 working draft
  • Supported by the following major browsers:
    Chrome 4+, FF 2+, IE 9+, Opera 9+, Safari 3.1+, iOS Safari 3.2+, Android Browser 2.1+, Opera Mobile 10.0+
  • Looks like: <canvas>…</canvas>

Canvas

  • Provides a low level raster based drawing API, meaning that you can manipulate the screen on a pixel by pixel basis.
  • Does NOT provide a scene graph. The canvas element has no memory of what was rendered previously, you have to track state yourself.
  • All user interaction has to be implemented by the programmer
       - No onclick or onmouseover! You have to implement these yourself
  • Provides methods for rendering images, gradients, and shadows, along with geometric primitives.

Canvas

The <canvas> element provides a getContext(string) method which fetches the drawing context to be used.
This allows the browser to support multiple drawing APIs or specifications.

Currently, two specifications exist:

2d
The W3C standard and most widely supported context
Provides a 2-D drawing API 
var context = canvasDomElement.getContext("2d");
WebGL
Experimental, only supported in WebKit and FireFox
Provides a 3-D draw API which mimics the OpenGL ES API 
var context = canvasDomElement.getContext("webgl");

Canvas

Code Sample!

		<body>
			<canvas id="myCanvas" width="500" height="500"></canvas>
			<script>
			var can = document.getElementById('myCanvas');
			var ctx = can.getContext('2d');
		
			ctx.fillStyle = "#006600";
			ctx.beginPath();
			ctx.arc(200, 200, 50, 0, Math.PI * 2); // draw circle
			ctx.fill();
			</script>
		</body>

Code Sample!

This draws the transparent Wikipedia logo on the canvas. 

		<body>
			<canvas id="myCanvas" width="500" height="500"></canvas>
			<script>
			var can = document.getElementById('myCanvas');
			var ctx = can.getContext('2d');
		
			var imageObj = new Image();

SVG (Scalable Vector Graphics)

Quick facts:

  • Born out of Microsoft's VML and Adobe's PGML
  • Open standard since 1999
  • Supported by the following major browsers (inline HTML):
    Chrome 7+, FF 4+, IE 9+, Opera 9+, Safari 5.1+, iOS Safari 5+, Android Browser 3.0+
  • Supported by the following major browsers (embed or object elements):
    Chrome 4+, FF 3+, IE 9+, Opera 9+, Safari 3.2+, iOS Safari 3.2+, Android Browser 3.0+, Opera Mobile and Opera Mini

SVG

  • XML file format...
    • Everything, including animations, are defined in XML
    • XML can be compressed easily, making for small files (.svgz)
    • Many images can be defined with SVG with less memory than a similar raster format (.png or .jpg)
  • In the browser: Retained mode rendering, similar to HTML
    • The browser "remembers" every node rendered, and builds a scene graph from XML
    • Every SVG node can be selected and its attributes altered, just like HTML
    • Transforms and other attributes apply to children elements

SVG

Code sample!

	<svg xmlns="http://www.w3.org/2000/svg" width="500px" height="500px">
		<circle cx="60" cy="60" r="40" fill="#006633" />
	</svg>

SVG

Code sample!

<svg xmlns="http://www.w3.org/2000/svg" width="600px" height="200px">
  <g>
    <circle cx="60" cy="60" r="40" fill="#006633" stroke-width="3" stroke="#00FF00" />
    <animateTransform attributeName="transform" type="scale" from="1" to="3" begin="0s" dur="5s" fill="freeze" />
  </g>
</svg>

How to Choose?

SVG Strengths

  • Content is searchable and can easily be made accessible
  • High detailed printing or viewing of content
  • Highly interactive content (think charts and graphs)
       - Browsers provide hitbox checking for you and fire appropriate events
  • Plain old CSS can be used to manipulate content

How to Choose?

Canvas Strengths

  • High volume drawing
  • Real time displaying of data
  • 3-D (WebGL)
  • Games
